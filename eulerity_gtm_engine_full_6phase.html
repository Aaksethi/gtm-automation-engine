Eulerity GTM Automation Engine

6-phase system · Google Sheets → Clay → AI Agent → Gmail → Lead Scoring → AI Reply → Sales Handoff

01
Email Validation
HubSpot export → ZeroBounce API → clean verified list → back to HubSpot
+
Goal: Before any email goes out, ensure the 8–10k contact database is clean. Dead emails damage domain reputation and waste enrichment credits. This phase runs once on the full database, then only on new incoming contacts.
📥
HubSpot API pull
GET · contacts endpoint
Paginated GET request to HubSpot Contacts API. Pulls in batches of 100. Extracts: email, first name, last name, company, source, create date, lifecycle stage. n8n Loop node handles all pages automatically.
→
⟺
Deduplicate + filter
n8n Code node
Removes duplicate emails. Filters contacts with no email address. Flags obvious fake patterns. Only unprocessed contacts with valid email format move forward.
→
✓
ZeroBounce Batch API
POST · bulk validate
Sends all emails to ZeroBounce Batch API in one call. Polls every 30s until job completes. Returns status per email: valid, invalid, catch-all, spamtrap, unknown. No emails are sent — this is a lookup only. Valid + catch-all proceed. Invalid + spamtrap are flagged.
→
📤
Write back to HubSpot
PATCH · contact property
Updates each HubSpot contact with custom property: email_verification_status = valid / invalid / catch-all. Invalid contacts: lifecycle stage → Unqualified. Valid contacts: tagged ready_for_enrichment = true. HubSpot stays the master record throughout.
02
Clay Enrichment + Segmentation
Valid contacts → Clay waterfall → company size, industry, franchise count → segmented → back to HubSpot
+
Goal: Transform a raw name + email into a full profile. Clay's waterfall searches 75+ data providers simultaneously — if Provider A has no data, it tries Provider B automatically. This gives the AI enough context to write emails that feel genuinely researched.
🔗
HubSpot → Clay
native integration
Clay's native HubSpot integration pulls all contacts where email_verification_status = valid. Creates a Clay table where each row = one contact. Pre-populates with existing HubSpot data: name, email, company, source.
→
🏢
Clay waterfall enrichment
75+ data providers
Enriches each contact across: job title, seniority, LinkedIn URL, company size, industry, HQ location, company revenue range, number of franchise locations, technology stack. Waterfall = if one provider fails, next tries automatically. Maximises fill rate on an 8-year-old database.
→
🏷
Segment classification
Clay AI column
Clay AI column classifies each contact: Segment A = decision maker at franchise brand with 50+ locations (hot ICP). Segment B = marketing manager at mid-size multi-location. Segment C = non-marketing title. Segment D = insufficient data. Each segment gets a different email sequence.
→
📤
Push back to HubSpot
Clay HubSpot sync
Clay pushes all enriched fields back to each HubSpot contact: job_title_enriched, company_size, industry, linkedin_url, segment_label, num_locations, enrichment_date. HubSpot is always the master record. Nothing lives in Clay long-term.
03
AI Personalized Email Generation + Send
HubSpot segments → Claude API → personalized email → HubSpot sends → tracks engagement
+
Goal: Every contact receives an email that feels like it was written specifically for them — referencing their company name, franchise scale, and industry pain. Claude generates each email from the enriched profile. HubSpot sends and tracks every interaction.
⏰
Weekly schedule trigger
n8n · every Monday 8am
Fires every Monday 8am. Pulls Segment A contacts from HubSpot where: segment_label = A AND last_email_sent_date is more than 7 days ago AND lifecycle_stage is not Customer. Batches into groups of 50 to respect API rate limits.
→
📝
Build personalization context
n8n Set node
Assembles a JSON context object per contact: first_name, company, job_title, company_size, industry, segment_label, num_locations, source. This becomes the Claude prompt input. The richer the context, the more specific the email.
→
🤖
Claude API — write email
claude-sonnet · JSON output
POST to api.anthropic.com/v1/messages. System prompt: senior Eulerity growth strategist, peer-to-peer tone, 3 paragraphs, no buzzwords, end with "would it make sense to connect for 20 minutes." User prompt passes full contact context.
→
✉
HubSpot Single Send API
transactional email · tracked
Sends via HubSpot's Single Send API — email comes from the sales rep's connected inbox, not a generic address. HubSpot automatically tracks: email opened, link clicked, replied, unsubscribed. All tracking data written to contact timeline.
04
Real-Time Lead Scoring + Routing
HubSpot webhooks → n8n → score updates → segment routing → sequence assignment
+
Goal: Every interaction a contact has with Eulerity's emails is tracked and scored in real time. Hot leads get faster follow-up. Cold leads stay in nurture. The system routes each contact to the right sequence based on their actual behavior — not assumptions.
🔔
HubSpot webhook
fires on email event
HubSpot sends real-time webhook to n8n whenever: email opened, link clicked, reply received. n8n Webhook Trigger listens at a public URL. Payload: contact ID, event type, timestamp, email ID, which URL was clicked.
→
🔢
Score increment logic
n8n Switch node
Routes by event type. Opened: +10 points. Link clicked: +25 points. Reply received: +50 points. Calendar link clicked: +100 points. Reads current score from HubSpot contact property, adds increment, writes back via PATCH API.
→
⟺
Score threshold routing
n8n IF node
Score 0–25: stays in cold nurture, no action. Score 26–75: warm tag, different email cadence. Score 76–149: hot tag, AI reply agent activates immediately. Score 150+: Sales Qualified Lead — immediate sales alert.
→
📊
HubSpot stage update
PATCH · lifecycle + score
Updates HubSpot contact: gtm_lead_score property = new score value. Lifecycle stage updates automatically based on threshold. All scoring data visible in HubSpot contact record and pipeline views. Sales team sees who is hot in real time.
05
AI Reply Agent
Reply received → classify intent → draft contextual response → send in same thread
+
Goal: When a contact replies to an Eulerity email, the AI reads the reply, understands the intent, and sends a contextual response — in the same email thread — within seconds. The agent has memory of the original email it sent, so every response feels like a natural continuation of the conversation.
📬
Reply webhook trigger
HubSpot · reply event
HubSpot detects incoming reply and fires webhook to n8n. Payload includes: reply text, contact ID, thread ID, message ID. n8n pulls the contact's full enriched profile and previous email thread from HubSpot timeline API to give the AI complete context.
→
🔍
Classify reply intent
Claude Haiku · fast + cheap
First Claude call — classification only. Prompt: classify this reply as exactly one of: INTERESTED / OBJECTION / NOT_NOW / UNSUBSCRIBE. Uses claude-haiku for speed and cost efficiency. Returns a single label. n8n Switch node routes based on that label.
→
✍
Draft contextual response
Claude Sonnet · memory-aware
Second Claude call — response drafting. Uses full context: thread history, contact enrichment data, intent label, Eulerity product details. INTERESTED: acknowledge, suggest specific next step. OBJECTION: address with relevant proof point. NOT_NOW: acknowledge timing, offer to reconnect in 30 days.
→
↩
Reply in same thread
HubSpot · same thread
AI-drafted reply sent via HubSpot — in the same email thread as the original. Contact sees it as a natural continuation of the conversation. Response time: under 15 minutes from reply received to reply sent. All replies logged in HubSpot contact timeline.
06
Sales Handoff + Post-Call Routing
Calendar booked → HubSpot deal created → sales briefing sent → post-call follow-up routed
+
Goal: When a contact books a sales call, the system takes over everything the sales rep would otherwise do manually — creates the deal, compiles a full briefing on the contact, notifies the right person, and routes the outcome after the call automatically.
📅
Calendly booking webhook
invitee.created event
Contact clicks calendar link in Eulerity email and books a slot. Calendly fires webhook to n8n immediately. Payload: contact email, meeting time, event type, pre-meeting form answers. n8n matches the email to the HubSpot contact record via search API.
→
💼
HubSpot deal created
Deals API · auto-populate
n8n creates Deal in HubSpot linked to contact and company. Deal name: "Sales Call — [Company] — [Date]". Pipeline stage: Meeting Scheduled. Deal auto-populates with all enrichment data, lead score, and meeting details. Contact lifecycle stage → Sales Qualified Lead.
→
📋
Sales briefing packet
Slack notification
n8n sends structured Slack message to assigned sales rep: contact name, company, title, lead score, segment, summary of the full email thread, what they responded to, pre-meeting form answers, HubSpot deal link, and meeting time. Rep walks in knowing the prospect — no manual research required.
